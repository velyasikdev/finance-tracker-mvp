# 1. Пітч та проблема

Більшість людей просто фіксують свої щоденні витрати, але не аналізують співвідношення між активами (те, що генерує дохід) та пасивами (те, що вимагає витрат), що заважає їм накопичувати капітал.

Наш вебзастосунок допомагає не лише вести облік транзакцій, а й чітко розділяти фінанси на активи та пасиви, формуючи мислення, орієнтоване на примноження багатства.

Цільова аудиторія - студенти, фрилансери та інвестори-початківці, які хочуть свідомо керувати своїми грошима.

# 2. MVP (Мінімально життєздатний продукт) та Ціль

Ціль першої ітерації: Створити базовий інтерфейс для ручного введення фінансових операцій та їх категоризації.

Ядро цінності (MVP): Один ключовий сценарій — можливість швидко додати запис, вказавши його тип (дохід, витрата, актив, пасив), і миттєво побачити оновлений загальний баланс на головному екрані.

# 3. Користувацький сценарій

* Користувач знаходиться на головній сторінці застосунку.

* 1. Натискає кнопку «Додати транзакцію».

  2. Вводить суму, обирає категорію (наприклад, «Актив: Депозит») і натискає «Зберегти».

* Новий запис з'являється в історії транзакцій, а діаграма загального капіталу автоматично оновлюється.

# 4. Картка стейкхолдерів

* Користувачі (Цільова аудиторія) сприйматимуть головну цінність продукту. Їхнє ключове очікування — інтерфейс має бути максимально простим, а час на внесення однієї транзакції не повинен перевищувати 5-10 секунд.

* Розробник відповідає за реалізацію технічної частини та оцінює досягнення цілей. Обмежений у часі рамками навчального семестру та дедлайнами здачі практичних робіт.

* Викладач оцінюватиме якість коду, архітектуру та відповідність результату поставленим вимогам.

# 5. Критерії успіху (Definition of Success) та Перевірка гіпотез

* Здатність нової людини додати актив чи пасив без додаткових підказок.

* Збір стислих відгуків від 3-5 тестових користувачів після першого використання, щоб перевірити, чи зрозуміла їм логіка розподілу на активи/пасиви.

# 6. Початковий перелік завдань

1. Спроєктувати базу даних для збереження транзакцій.

2. Розробити бекенд-логіку для обробки CRUD-операцій.

3. Створити базовий веб-інтерфейс (форму вводу та таблицю виводу).
